### How can I ensure that my code, program, or software is functional and secure?
> Functional and secure coding is a pinnacle of software development. In order to ensure these qualities, I take a disciplined approach to coding, utilizing established patterns and appropriate testing and validation techniques. Throughout this project, I used JUnit 5 to verify each component behaved as expected under various conditions. By confirming both valid and invalid input, I avoided introducing bias to the best of my ability, and made sure the program produced accurate results, or failed in a controlled and predictable way. I also used coverage metrics as a way to validate that my tests were exercising all execution paths.    
> Beyond testing, I enforced strict input validation within the classes themselves, ensuring that invalid data was rejected immediately. This combination of proactive validation and structured testing helped create software that is both reliable and resistant to unintended behavior.
### How do I interpret user needs and incorporate them into a program?
> Treating user needs as enforceable requirements helps translate them directly into program logic, such as limiting field lengths, non-null values, and ensuring dates are not set in the past. From this point, I implemented and tested those rules through targeted test cases to confirm they aligned with the desired functionality and behavior. This is also a key point to consider boundary conditions and edge cases, since this is often where assumptions and technical debt can creep in. This approach allowed me to move beyond simply implementing features and instead build software confidently from the user's perspective.
