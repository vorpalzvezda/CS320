# CS320
Collection of work from CS320

### How can I ensure that my code, program, or software is functional and secure?
> Functional and secure coding is a cornerstone of effective software development. To ensure these qualities, I take a disciplined approach, utilizing established design practices along with appropriate testing and validation techniques. Throughout this project, I used JUnit 5 to verify that each component behaved as expected under a variety of conditions. By testing both valid and invalid inputs, I reduced bias and ensured the program either produced accurate results or failed in a controlled and predictable way. I also used coverage metrics to confirm that my tests exercised the majority of executions paths.  
> Beyond testing, I enforced strict input validation within the classes themselves, ensuring that invalid data is rejected immediately. This combination of proactive validation and structured testing results in software that is both reliable and ressistant tto unintended behavior. 
### How do I interpret user needs and incorporate them into a program?
> Treating user needs as enforceable requirements helps translate them directly into program logic, such as limiting field lengths, non-null values, and ensuring dates are not set in the past. From this point, I implemented and tested those rules through targeted test cases to confirm they aligned with the desired functionality and behavior. This is also a key point to consider boundary conditions and edge cases, since this is often where assumptions and technical debt can creep in. This approach allowed me to move beyond simply implementing features and instead build software confidently from the user's perspective. 
### How do I approach designing software?
> I try to take a slow and methodical approach to software design, prioritizing structure, clarity, and reusability. I organized this program by separating data representations from operational logic, using dedicated classes for each and service layers to manage object interactions. Not only does this improve readability, it allows individual components to be modified without impacting functionality of other parts of the program. I also tried to minimize redundancy by centralizing common logic, which helped improve efficiency and consistency.
> Overall, I took an iterative approach. Beginning with simple implementation that satisfies requirements, then refine each component through testing and revision to produce a reliable design.
